Resumen de Códigos de Respuesta HTTP

Los códigos de respuesta HTTP indican el estado de una solicitud. Se agrupan en cinco categorías:

  1. Respuestas informativas (1xx)

    100 Continue: El cliente puede continuar con la solicitud.

    101 Switching Protocols: El servidor acepta un cambio de protocolo.

    102 Processing: El servidor sigue procesando la solicitud.

  2. Respuestas satisfactorias (2xx)

    200 OK: Solicitud completada con éxito.

    201 Created: Recurso creado exitosamente.

    204 No Content: Solicitud exitosa, sin contenido.

  3. Redirecciones (3xx)

    301 Moved Permanently: El recurso se movió permanentemente.

    302 Found: Recurso movido temporalmente.

    304 Not Modified: Sin cambios, útil para caché.

  4. Errores del cliente (4xx)

    400 Bad Request: Solicitud mal formada.

    401 Unauthorized: Se requiere autenticación.

    404 Not Found: Recurso no encontrado.

  5. Errores del servidor (5xx)

    500 Internal Server Error: Error interno en el servidor.

    503 Service Unavailable: Servidor sobrecargado o en mantenimiento.

    504 Gateway Timeout: Tiempo de espera excedido.
