Estructura y funcionamiento

El Protocolo de Transferencia de Hipertexto es un protocolo utilizado en la World Wide Web para la transferencia de datos, especialmente documentos de hipertexto, como páginas web. Mantiene una estructura basada en los clientes y servidores y orientada a transacciones.

  1. Cliente: El cliente es el software o la aplicación que solicita recursos (como páginas web) desde un servidor web. El cliente inicia la comunicación HTTP enviando una solicitud al servidor.
  2. Servidor: El servidor es el software que almacena los recursos web y los entrega a los clientes cuando se les solicita. Responde a las solicitudes HTTP que recibe de los clientes.
  3. URI (Uniform Resource Identifier): La URI es una cadena que identifica un recurso en la web. Es parte fundamental de una solicitud HTTP y puede ser una URL (Uniform Resource Locator) que incluye la dirección del recurso y el protocolo a utilizar.
  4. Método HTTP: La solicitud HTTP incluye un método que indica la acción que se debe realizar en el recurso identificado por la URI.
  5. Encabezados HTTP: Los encabezados son metadatos que proporcionan información adicional sobre la solicitud o la respuesta. Pueden incluir información sobre el tipo de contenido, la longitud del contenido, cookies, entre otros.
  6. Cuerpo del mensaje: En algunas solicitudes y respuestas, se incluye un cuerpo que contiene datos. Por ejemplo, en una solicitud POST, el cuerpo puede contener datos que el servidor debe procesar.
  7. Código de estado HTTP: En las respuestas HTTP, se incluye un código de estado que indica el resultado de la solicitud. Algunos códigos de estado comunes son:
    • 200 OK: La solicitud se ha completado con éxito.
    • 404 Not Found: El recurso solicitado no fue encontrado en el servidor.
    • 500 Internal Server Error: El servidor encontró un error interno al procesar la solicitud.
  8. Respuesta HTTP: La respuesta del servidor incluye el código de estado, encabezados y, en muchos casos, un cuerpo que contiene el recurso solicitado o información adicional.
